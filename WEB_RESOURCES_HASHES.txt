# WEB_RESOURCES_HASHES.txt
#
# SHA-256 hashes of every vendored third-party asset in GPXEditor/WebResources/.
# Per SECURITY.md "Vendored web assets" and Docs/03_WEB_RESOURCES.md, vendored
# files are hash-pinned here.  A pre-commit hook (or CI step) recomputes the
# hashes of staged files and rejects divergence unless this file is updated
# in the same commit with a message naming the upstream version.
#
# Project-authored files (index.html, editor.js, editor.css) are NOT listed
# here — their authoritative version is the repository content itself.
#
# Lines beginning with `#` and blank lines are comments.  The non-comment
# lines are in `shasum -a 256 -c` format:  64-hex SHA-256, two spaces, path
# relative to the repository root.  Verify locally from the repository root:
#
#     shasum -a 256 -c WEB_RESOURCES_HASHES.txt
#
# Updating a vendored file follows the protocol in SECURITY.md "Update
# protocol for vendored files":  verify upstream legitimacy, download via
# HTTPS, diff and scan for red flags, replace the vendored file, recompute
# the hash, update the comment block above the entry to name the new
# upstream version, commit both the asset change and this file together.
